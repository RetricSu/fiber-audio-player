Migration System Features:
--------------------------
- Auto-runs pending migrations on backend startup
- Transaction support (atomic via better-sqlite3 db.transaction())
- Version tracking via migrations table (id, name, applied_at)
- Natural sort ordering (001_, 002_, etc.)
- Idempotent - skips already applied migrations

Migrations Table Schema:
------------------------
CREATE TABLE migrations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  applied_at INTEGER NOT NULL DEFAULT (CAST(strftime('%s', 'now') AS INTEGER) * 1000)
);
