Features Implemented:
--------------------
- Process uploaded MP3 files with FFmpeg
- Generate HLS with 6-second segments (configurable, default: 6s)
- Apply AES-128 encryption using randomly generated keys
- Normalize loudness to -16 LUFS using FFmpeg loudnorm filter
- Extract duration metadata using ffprobe
- Update episode status: draft → processing → ready
- Store HLS files in episode directory (uploads/{podcast_id}/{episode_id}/hls/)
- Handle FFmpeg errors gracefully with cleanup

HLS Output Structure:
---------------------
uploads/{podcast_id}/{episode_id}/
├── source.mp3 (original upload)
└── hls/
    ├── playlist.m3u8 (master playlist)
    ├── enc.key (AES-128 encryption key)
    └── segment_001.ts, segment_002.ts, ... (encrypted segments)

FFmpeg Configuration:
---------------------
- Audio codec: AAC
- Audio bitrate: 128k
- Segment duration: 6 seconds
- Playlist type: VOD (Video on Demand)
- Loudness normalization: -16 LUFS, -1.5 dBTP, 11 LRA
- Encryption: AES-128 with random 16-byte key and IV

Queue Configuration:
--------------------
- Concurrency: 2 jobs concurrently
- Retry attempts: 3
- Retry delay: 5000ms

Episode Status Workflow:
------------------------
draft → processing → ready → published

APIs Available:

Dependencies:
-------------
- FFmpeg (must be installed on system)
- ffprobe (included with FFmpeg)
- Node.js child_process for FFmpeg execution

Security Notes:
---------------
- Encryption key info file (enc.keyinfo) is deleted after transcoding
- Only enc.key and encrypted segments are retained
- Random 16-byte keys are generated per episode
