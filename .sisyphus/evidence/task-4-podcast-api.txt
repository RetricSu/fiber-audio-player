# Task 4: Admin API Endpoints for Podcast CRUD

## Implementation Evidence

Date: 2026-03-07

### Endpoints Implemented

1. **POST /admin/podcasts** - Create podcast
2. **GET /admin/podcasts** - List all podcasts
3. **GET /admin/podcasts/:id** - Get podcast details
4. **PUT /admin/podcasts/:id** - Update podcast
5. **DELETE /admin/podcasts/:id** - Delete podcast (cascade delete episodes)

### Features Implemented

- Simple API key authentication using `ADMIN_API_KEY` env variable
- Bearer token format: `Authorization: Bearer {key}`
- Input validation:
  - Title is required for creation
  - Title max length 255 characters
  - Returns 400 for validation errors
  - Returns 401 for auth failures
  - Returns 404 for not found
  - Returns 201 for successful creation
- UUID generation using `crypto.randomUUID()`
- JSON request/response handling with `{ ok: true/false, ... }` format
- Cascade delete for episodes via SQLite `ON DELETE CASCADE` foreign key constraint

### Example Usage

```bash
# Set API key
export ADMIN_API_KEY="your-secret-key"

# Create podcast
curl -X POST http://localhost:8787/admin/podcasts \
  -H "Authorization: Bearer your-secret-key" \
  -H "Content-Type: application/json" \
  -d '{"title": "My Podcast", "description": "A great podcast"}'

# List podcasts
curl http://localhost:8787/admin/podcasts \
  -H "Authorization: Bearer your-secret-key"

# Get podcast details
curl http://localhost:8787/admin/podcasts/abc-123 \
  -H "Authorization: Bearer your-secret-key"

# Update podcast
curl -X PUT http://localhost:8787/admin/podcasts/abc-123 \
  -H "Authorization: Bearer your-secret-key" \
  -H "Content-Type: application/json" \
  -d '{"title": "Updated Title"}'

# Delete podcast (cascades to episodes)
curl -X DELETE http://localhost:8787/admin/podcasts/abc-123 \
  -H "Authorization: Bearer your-secret-key"
```
